# Introduction

## 1. 개요 (Overview)
LAYOUTNEMO 프로젝트는 기존의 리스트 및 보드 중심의 업무 관리 도구가 담아내지 못했던 사고의 흐름과 업무 간 관계를 시각적으로 펼쳐놓기 위한 개인 업무 사고 공간을 제공합니다. 이 프로젝트는 업무를 관리하는 도구가 아니라, 업무를 어떻게 인식하고 있는지를 외부에 드러내는 공간을 만드는 데서 출발했습니다. {: .label .label-blue }

## 2. 아키텍처 및 로직 (Architecture & Logic)
LAYOUTNEMO는 사용자의 사고 흐름을 시각적으로 표현하기 위해 캔버스 기반의 인터페이스를 제공합니다. 사용자는 블럭을 생성하고, 영역을 통해 사고의 맥락을 분리하며, 블럭 간의 관계를 시각적으로 연결할 수 있습니다. 이러한 구조는 사용자가 사고를 정리하기 전에 그 상태 그대로 펼쳐놓을 수 있도록 설계되어 있습니다.

## 3. 핵심 컴포넌트 분석 (Key Components)

### 3.1 주요 함수/클래스
- **설명**: LAYOUTNEMO의 주요 컴포넌트는 블럭과 영역을 관리하고, 사용자의 인터랙션을 처리하는 데 중점을 두고 있습니다.
- **상세 명세 (테이블)**:

  | 파라미터 | 타입 | 설명 |
  |:--------|:-----|:-----|
  | blocks  | WorkBlock[] | 사용자 작업 블럭 목록 |
  | zones   | Zone[] | 사고 맥락을 분리하는 영역 목록 |
  | selectedZone | string | 현재 선택된 영역 |
  | showRelationships | boolean | 블럭 간 관계선 표시 여부 |
  | showCompletedBlocks | boolean | 완료된 블럭 표시 여부 |

### 3.2 데이터 구조
- **설명**: 데이터 구조는 사용자의 사고 흐름을 반영하도록 설계되어 있으며, 블럭과 영역의 상태를 저장하고 관리합니다.
- **상세 명세 (테이블)**:

  | 필드명 | 타입 | 제약조건 | 설명 |
  |:------|:----|:--------|:----|
  | id    | string | 고유값 | 블럭 또는 영역의 식별자 |
  | title | string | 필수 | 블럭의 제목 |
  | description | string | 선택 | 블럭의 설명 |
  | x, y  | number | 필수 | 블럭의 위치 좌표 |
  | width, height | number | 필수 | 블럭의 크기 |

## 4. 사용 예시 (Usage)
LAYOUTNEMO는 사용자가 블럭을 생성하고, 이를 다양한 영역에 배치하여 사고의 흐름을 시각적으로 표현할 수 있도록 합니다. 예를 들어, '사용자 인터뷰 진행' 블럭을 '기획' 영역에 배치하고, '프로토타입 개발' 블럭과 연결하여 작업의 흐름을 나타낼 수 있습니다.

## 5. 설정 (Configuration)
- **환경 변수**: LAYOUTNEMO는 로컬 저장소를 통해 사용자의 작업 상태를 유지하며, 다음과 같은 키를 사용합니다.

  | 환경 변수 | 설명 |
  |:---------|:----|
  | STORAGE_KEY | 저장된 캔버스 데이터의 키 |
  | CURRENT_CANVAS_KEY | 현재 선택된 캔버스의 키 |
